# My_PID_Motor v2.1 负载扰动测试计划

> 当前文档只是测试计划，不代表测试已经执行。

## 1. 共用条件与安全边界

当前源码对 UART 非零目标的接受范围为绝对值 `500～1000`，现有调参记录覆盖 `±500、±600、±800、±1000`。因此本计划选择正向 500、600、800 作为低、中、较高目标的建议值；它们是基于现有范围确定的计划参数，不是 v2.1 已完成结果。

三组测试统一使用：

- UART target；
- `Kp=0.05, Ki=0, Kd=0`；
- 真实 CRLF；
- LLCOM 或等价工具保存完整 TX/RX、时间戳和本地 MARK；
- 约每 300 ms 主动发送一次 `status`，不修改固件默认日志周期；
- 正向单向测试，每组结束后先 stop，不做正反转直接切换；
- 人工轻微负载或安全外部负载装置，不堵转，不用手直接接触高速旋转件。

如出现异常声音、明显发热、剧烈抖动、失控或 fault，应立即 stop。若出现 fault，在确保安全后保存当前 status 和 `get fault`，不要先用 `rst` 清除证据。

## 2. LD-001：低目标速度负载扰动

- **测试编号**：LD-001
- **测试目标**：观察当前最低支持 UART 目标附近，轻载扰动是否引起 actual 下降和 PWM 补偿，以及释放后能否恢复持续运行。
- **前置条件**：确认驱动板、电机供电和编码器连接正常；电机可在 500 目标下无负载持续运行；status 显示 `fault:NONE`、`kp:0.050`、`ki:0.000`、`kd:0.000`。
- **建议 target**：500。该值等于当前源码的非零最小接受绝对值，并已有持续运行记录；上板前仍需确认本次电机、驱动和供电未变化。
- **扰动方式**：稳定运行约 3 秒后，通过安全负载装置轻微增加阻力约 1.5～2 秒，再完全释放并观察约 3 秒。禁止堵转。
- **需要记录的字段**：`ms、enable、State、source、target、actual、delta、PWM、fault、kp、ki、kd`；`adc1/adc2` 保留在原始日志中。
- **观察重点**：500 接近当前持续运行下限，重点看负载时是否停转、PWM 是否提高、释放后是否重新稳定，而不是追求固定的瞬时下降幅度。
- **通过判据**：扰动前可稳定运行；轻载时 actual 有可解释下降且 PWM 相对基线上升；释放后 actual/PWM 回到扰动前附近；全程无异常 fault；最终 stop 后 `enable:0、State:IDLE、PWM:0`。
- **失败判据**：轻微负载下立即持续停转且释放后不恢复；actual 下降但 PWM 没有可解释变化；状态或目标源意外改变；出现 fault、复位、失控或 stop 后未安全归零。
- **原始日志保存路径**：`logs/v2.1_LD-001_raw.txt`
- **报告保存路径**：`reports/v2.1_LD-001_report.md`
- **注意事项**：该档最容易受静摩擦和负载差异影响。若人工负载明显过大，应将该次标记为“条件不可比”，不能直接判定控制失败。

建议串口步骤：

```text
stop
status
set target uart
kp=0.05
ki=0
kd=0
t=500
run=1

等待约 3 秒并持续采集 status
MARK: APPLY_LOAD
施加轻微负载约 1.5～2 秒
MARK: RELEASE_LOAD
释放负载并等待约 3 秒

stop
status
```

## 3. LD-002：中目标速度负载扰动

- **测试编号**：LD-002
- **测试目标**：在已有 `target=600` 人工轻载结果基础上，为 v2.1 候选固件形成独立、可追溯的中速负载扰动证据。
- **前置条件**：LD-001 已安全结束；硬件连接和供电未改变；status 显示 `fault:NONE`，PID 参数与本计划一致。
- **建议 target**：600。仓库已有该档轻载趋势记录，可用于对比，但旧记录不等同于本轮已经执行。
- **扰动方式**：稳定运行约 3 秒，使用与 LD-001 相同的安全方式施加轻微负载约 1.5～2 秒，释放后观察约 3 秒。
- **需要记录的字段**：`ms、enable、State、source、target、actual、delta、PWM、fault、kp、ki、kd`，并完整保留 `adc1/adc2`。
- **观察重点**：actual 下降时 PWM 是否同步高于扰动前基线；释放后两者是否回到 600 档基线附近；与旧 `target=600` 记录只做趋势对照，不强求完全相同数值。
- **通过判据**：状态和 target 保持不变；负载期间 actual 下降、PWM 相对提高；释放后恢复到扰动前附近；`fault:NONE`；stop 后软件状态归零。
- **失败判据**：PWM 不响应 actual 变化；释放轻载后长时间无法恢复；出现不可解释的 actual/PWM 跳变、fault、复位、串口失联或 stop 失效。
- **原始日志保存路径**：`logs/v2.1_LD-002_raw.txt`
- **报告保存路径**：`reports/v2.1_LD-002_report.md`
- **注意事项**：人工负载大小和动作时刻无法精确测量，本组只能支持定性趋势判断，不能用于计算恢复时间、带宽或严格动态指标。

建议串口步骤：

```text
stop
status
set target uart
kp=0.05
ki=0
kd=0
t=600
run=1

等待约 3 秒并持续采集 status
MARK: APPLY_LOAD
施加轻微负载约 1.5～2 秒
MARK: RELEASE_LOAD
释放负载并等待约 3 秒

stop
status
```

## 4. LD-003：较高目标速度负载扰动

- **测试编号**：LD-003
- **测试目标**：观察较高但未到软件上限的目标下，轻载扰动期间软件 PWM 命令和速度反馈是否仍呈现可解释的补偿趋势。
- **前置条件**：前两组没有留下未处理 fault、异常发热或机械问题；确认旋转部件防护和负载装置适合更高转速。
- **建议 target**：800。该值位于当前已验证运行范围内并低于 1000 上限，可作为较高目标代表；若现场安全条件不足，应停止本组并记录“未执行”。
- **扰动方式**：稳定运行约 3 秒后，用安全负载装置轻微增加阻力约 1.5～2 秒，完全释放后观察约 3 秒。不得徒手接触旋转部件。
- **需要记录的字段**：`ms、enable、State、source、target、actual、delta、PWM、fault、kp、ki、kd`，原始日志保留 `adc1/adc2`。
- **观察重点**：PWM 是否有剩余调节空间、actual 是否在释放后回到原基线附近，以及是否出现 `PWM_SATURATION` 或其他 fault。
- **通过判据**：负载期间 actual/PWM 形成可解释的反向变化，释放后恢复；没有异常 fault、失控或串口中断；stop 后安全归零。
- **失败判据**：轻微扰动即出现持续失速、PWM 异常饱和、fault、机械异常或释放后不恢复；软件显示与硬件现象明显冲突；stop 后输出未归零。
- **原始日志保存路径**：`logs/v2.1_LD-003_raw.txt`
- **报告保存路径**：`reports/v2.1_LD-003_report.md`
- **注意事项**：更高转速意味着更高机械风险。本组是否执行以现场防护为先；未执行时如实记录，不用低档结果外推。

## 5. 结果填写原则

- 计划中的“通过判据”是后续判断规则，不是当前结果；
- 原始日志必须来自真实串口输出，不得将示例、预期或 AI 推测写入日志；
- 报告必须引用对应日志、固件 commit 和构建产物 hash；
- 没有示波器、逻辑分析仪或电流测量时，相关物理量写“未测，不作为本轮结论”；
- 人工负载结果只描述相对趋势，不宣称精确恢复时间、控制带宽或长期可靠性。
