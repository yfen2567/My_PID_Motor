# My_PID_Motor status 日志字段表

## 1. 适用范围

本文依据当前 `LogTask.c` 的 status 输出格式和现有诊断记录整理，只描述实际存在的字段：

```text
ms,enable,State,source,target,actual,delta,PWM,adc1,adc2,fault,kp,ki,kd
```

字段值是软件状态和采样结果，不能单独证明物理 PWM 波形、方向引脚电平、电机电流或编码器电气波形正确。

## 2. 字段字典

| 字段 | 含义 | 单位或取值范围 | 正常情况下怎么看 | 负载扰动诊断价值 | 故障定位价值 |
| --- | --- | --- | --- | --- | --- |
| `ms` | `HAL_GetTick()` 返回的开机后时间 | ms，32 位无符号计数，会回绕 | 连续日志应总体递增；结合串口本地时间判断阶段 | 对齐施加载荷、释放载荷和恢复阶段；只能提供日志时间尺度 | 可定位异常发生的相对时间，不能单独说明原因 |
| `enable` | 控制运行使能 | `0` 或 `1` | IDLE/stop 时通常为 0，RUN 时为 1 | 确认扰动期间控制器仍处于使能状态 | 可判断保护或 stop 是否取消了运行使能 |
| `State` | 软件状态机状态 | `IDLE`、`RUN`、`FAULT`、`CALIB` | 正常负载测试应在 RUN；结束后回到 IDLE | 防止把状态切换造成的转速变化误判为负载响应 | FAULT 可提示进入保护；需结合 `fault` 与快照 |
| `source` | 当前目标速度来源 | `Uart` 或 `ADC` | UART 测试应持续显示 `Uart` | 确认 target 没有被 ADC 输入变化替换 | 可排除目标来源切换导致的异常 |
| `target` | 当前目标速度 | RPM；UART 解析上限 `±1000`，当前非零最小接受绝对值为 500 | 应与已发送的目标一致，符号表示方向 | 作为 actual 和 PWM 变化的参照 | 可判断异常是否伴随目标突变；不能证明命令执行时序 |
| `actual` | 编码器窗口计算得到的实际速度 | RPM，有符号；受约 42.9 RPM 测速量化影响 | 稳态时在 target 附近跳动，方向符号应匹配 | 核心字段：观察负载时下降幅度及释放后是否恢复 | 可发现停转、反向、超速或反馈异常，但需结合 `delta` |
| `delta` | 最近一个控制周期的编码器计数增量 | encoder count / 10 ms，有符号整数 | 转动时应随方向有相应符号；停转时接近 0 | 辅助判断 actual 下降是否来自编码器脉冲减少 | 对 actual=0、方向异常和编码器丢失有价值；不是完整原始波形 |
| `PWM` | 软件输出的有符号 PWM 命令 | `-1000～1000`，不是百分比；符号表示方向 | stop 时应为 0；运行时应与方向和控制需要一致 | 核心字段：actual 下降时 PWM 是否相对基线提高 | 可判断软件是否在尝试输出；不能代替 PA8 实测波形 |
| `adc1` | ADC 主通道原始值 | 12 位 ADC count，通常 `0～4095` | UART 目标源下允许变化，但不应改变 target | UART 负载测试中主要用于排除异常 ADC 波动与控制现象混淆 | ADC 源问题的辅助证据；不是电压值，除非另行换算 |
| `adc2` | ADC 辅助通道原始值 | 12 位 ADC count，通常 `0～4095` | 记录其稳定性，不预设物理含义 | 仅作同期辅助采样，不作为负载补偿主判据 | 可提示辅助输入变化；当前 fault snapshot 的 `adc_aux` 赋值完整性另有未确认项 |
| `fault` | 当前软件故障码 | `NONE`、`SPEED_OVER_LIMIT`、`ENCODER_LOST`、`PWM_SATURATION` | 正常验证应为 `NONE` | 判断扰动是否触发保护；出现非 NONE 应立即结束并保存证据 | 可做故障粗分类，根因仍需 status、快照和硬件现象共同判断 |
| `kp` | 当前比例系数 | 无量纲；命令接受范围 `0～100`，输出保留 3 位小数 | 默认应显示 `0.050` | 确认扰动补偿条件没有被在线命令改变 | 可排除参数错误，但不能单独定位硬件故障 |
| `ki` | 当前积分系数 | 无量纲；命令接受范围 `0～100`，输出保留 3 位小数 | 当前默认应显示 `0.000` | 确认本轮不含积分作用 | 可排除积分参数变化或累积影响 |
| `kd` | 当前微分系数 | 无量纲；命令接受范围 `0～100`，输出保留 3 位小数 | 当前默认应显示 `0.000` | 确认本轮不含微分作用 | 可排除微分参数变化，不能代替噪声分析 |

## 3. 负载扰动时的最小联合判断

负载扰动不能只看单个字段，至少同时观察：

1. `enable=1`、`State:RUN`、`source:Uart`、`target` 不变；
2. 施加载荷后 `actual` 是否下降，`delta` 是否同步反映编码器计数减少；
3. `PWM` 是否相对扰动前基线提高；
4. 释放载荷后 `actual` 和 `PWM` 是否回到原基线附近；
5. 全程 `fault:NONE`，结束 stop 后 `enable:0`、`State:IDLE`、`PWM:0`。

如果 `PWM` 上升但 actual 不恢复，只能说明软件提高了输出命令，不能由 status 单独区分负载过大、供电/驱动能力不足、机械阻力或物理 PWM 输出异常。

## 4. fault snapshot 与 status 的边界

当前 fault snapshot 输出字段为：

```text
valid,time,state,fault,T_speed,A_speed,pwm,adc_target,adc_aux
```

仓库已有证据只确认了无快照时返回 `FAULT_SNAPSHOT_NONE`。真实故障状态下的快照字段完整性仍待验证；其中 `adc_aux` 在现有字段评审中被标记为当前保存路径未赋值，不能把它当成已验证测量值。
