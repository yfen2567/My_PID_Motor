# target=600 负载扰动测试结果

## 一、测试条件

- target source：UART
- target：600
- Kp：0.05
- Ki：0
- Kd：0
- 串口日志：`llcom_load_disturbance_600_log.txt`
- 负载方式：人工轻微负载扰动
- 采样方式：status 自动采样
- 阶段记录：MARK 分段记录

原始日志当前位于外部路径 `D:\llcom_load_disturbance_600_log.txt`，未归档进真实工程或 Codex 工作区。本文件根据该真实串口日志整理结果，不包含重复测试或推测结果。

## 二、分阶段结果

| 阶段 | actual 均值 | 最低 actual | PWM 均值 | 最高 PWM |
| ----- | ----------: | ----------: | -------: | -------: |
| 扰动前 | 604.7 | — | 127.5 | — |
| 第一次负载 | 580.7 | 471 | 128.7 | 134 |
| 第一次释放 | 594.7 | — | 128.1 | — |
| 第二次负载 | 568.8 | 385 | 129.3 | 138 |
| 第二次释放 | 601.5 | — | 127.7 | — |

## 三、结果结论

- 两次负载期间 actual 均下降；
- 两次负载期间 PWM 均同步提高；
- 第二次负载更明显，actual 最低降至 385，PWM 最高升至 138；
- 负载释放后 actual 均恢复到 600 附近；
- 负载释放后 PWM 恢复到约 128 的基线附近；
- 全程 `fault:NONE`；
- 最终 stop 后为 `enable:0、State:IDLE、PWM:0、actual:0`；
- 因此，在 `Kp=0.05、Ki=0、Kd=0` 条件下，系统具备基本轻载扰动补偿能力。

## 四、结论边界

- 本次负载由人工施加，负载大小和动作时刻没有精确测量；
- 本日志不能用于计算精确恢复时间、控制带宽或严格动态指标；
- 本测试只能支持定性结论：轻载扰动下，actual 下降时 PWM 会提高，释放后能够恢复；
- 本测试无需再次执行。
