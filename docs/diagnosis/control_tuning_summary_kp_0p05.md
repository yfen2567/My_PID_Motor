# My_PID_Motor v2.0 后续控制调参记录：零 PID 前馈基线与 Kp=0.05 全范围验证

## 一、背景

本阶段发生在 `v2.0-rtos-cmdqueue` 发布基线之后，属于控制效果调参与真实运行现象验证，不再属于命令压力测试或 release 封版流程。

本阶段不增加功能，主要验证：

- 修正后的前馈是否在正反向全范围有效；
- `Kp=0.05、Ki=0、Kd=0` 是否适合作为默认 PID 参数；
- 当前是否有必要继续引入 Ki 或 Kd。

## 二、测试 1：零 PID 全量前馈基线

测试条件为 `Kp=0、Ki=0、Kd=0`，依次验证正反向 `500、600、800、1000` 四个目标档位。

| target | PWM | 稳态 actual 均值 |
| -----: | --: | ----------------: |
| 500 | 115 | 约 457 |
| 600 | 128 | 约 586 |
| 800 | 153 | 771 |
| 1000 | 178 | 985 |
| -500 | -115 | 约 -471 |
| -600 | -128 | 约 -586 |
| -800 | -153 | -771 |
| -1000 | -178 | 约 -974 |

测试结论：

- 八个档位均持续运行；
- 电机与编码器方向正确；
- 全程 `fault:NONE`；
- 未出现 `ERR`；
- 正反向 600、800 档基本镜像；
- 反向启动助推方向修复有效；
- 最终执行 stop 后为 `enable:0、State:IDLE、PWM:0、actual:0`；
- 当前测速分辨率约为 42.9 RPM，`428/471` 等离散跳动包含编码器量化影响，不能要求每次状态输出完全等于 target。

## 三、测试 2：Kp=0.05，target=±600 验证

测试条件为 `Kp=0.05、Ki=0、Kd=0`，剔除启动阶段后统计稳态数据。

| 方向 | actual 均值 | actual 范围 | PWM 范围 |
| ---- | ----------: | ----------: | -------: |
| 600 | 599.9 | 557～642 | 125～130 |
| -600 | -613.9 | -642～-557 | -130～-125 |

测试结论：

- 闭环控制方向正确；
- 正向 24 个稳态样本中有 18 个为 `actual=600`；
- 反向略有超调，均值偏差约 14 RPM，小于一个约 42.9 RPM 的测速量化单位；
- PWM 只在基准值 128 附近调整约 ±3；
- 未观察到明显振荡；
- 全程 `fault:NONE`；
- 最终 stop 后安全归零；
- `Kp=0.05` 可以保留，不建议继续增大；
- 当前暂不需要加入 Ki 或 Kd。

## 五、默认 PID 参数决策

当前建议并已采用的默认参数为：

```c
#define APP_PID_DEFAULT_KP 0.05f
#define APP_PID_DEFAULT_KI 0.0f
#define APP_PID_DEFAULT_KD 0.0f
```

- `Kp=0.05` 基于 `±500、±600、±800、±1000` 全范围实测结果确定；
- `Ki=0` 是因为当前稳态误差已经小于或接近测速量化粒度，继续增加积分的收益不明确，反而可能引入低速抖动或积分累积问题；
- `Kd=0` 是因为当前未观察到明显振荡或过冲问题，暂时没有加入微分的必要。

## 九、负载扰动验证补充

在 `target=600、Kp=0.05、Ki=0、Kd=0` 条件下完成了两次人工轻微负载扰动。两次扰动均表现为 actual 下降、PWM 同步上升，释放负载后 actual 和 PWM 恢复到基线附近，且全程 `fault:NONE`。该结果进一步支持 `Kp=0.05` 作为当前默认 Kp。

本次测试属于人工轻载条件下的定性验证，不用于计算精确恢复时间、控制带宽或严格动态指标。详细结果见 `docs/diagnosis/load_disturbance_600_result.md`。
