# 调试记录

调试记录用于记录项目如何从“不稳定”逐步调到“稳定”。每次记录只写一个测试目标，不要把多个问题混在一起。

调试记录不是作文，也不是项目介绍。它的重点是留下证据：当时发了什么命令、串口输出是什么、现象是什么、下一步准备怎么判断。

### 第 3 次调试：确认速度单位和测速窗口影响

| 项目 | 内容 |
|---|---|
| 日期 | 2026-06-09 晚上 |
| 测试目标 | 把编码器计数换算成 RPM，并确认 10ms / 50ms 测速窗口对低速反馈的影响 |

### 第 4 次调试：定位双相编码器启动异常

| 项目 | 内容 |
|---|---|
| 日期 | 2026-06-10 |
| 测试目标 | 解决两相编码器接入后，一 `run=1` 就 `SPEED_OVER_LIMIT` / `PWM_SATURATION` 的问题 |
| 当前代码版本 | TIM2 编码器模式改为 `TIM_ENCODERMODE_TI12`，速度按输出轴 7 脉冲、x4 解码估算为一圈 28 count |
| 硬件状态 | A/B 两相接入 PA15/PB3，后改为 GPIO 上拉 |
| 串口命令 | `rst`、`set target uart`、`t=300`、`kp=0.5~0.9`、`ki=0`、`kd=0`、`run=1` |
| PID 参数 | `kp=0.5~0.9, ki=0, kd=0` |
| 目标速度 | `target=300 RPM` |
| 实际现象 | 两相都接入时启动立刻超速或饱和；拔掉 A/B 其中一根线后现象明显改善；将 PA15/PB3 都改成 `GPIO_PULLUP` 后，两相同时接入可以正常运行 |
| 串口输出摘录 | 异常时：`enable:0,State:FAULT,source:Uart,target:300,actual:0,delta:0,PWM:0,fault:SPEED_OVER_LIMIT,kp:0.80,ki:0.00,kd:0.00`；另一次 ADC 源下出现 `fault:PWM_SATURATION` |
| 问题判断 | 问题根因不是 PID 参数，而是编码器 A/B 输入在无上拉时容易受浮空或干扰影响，两相信号同时参与解码时产生异常计数；故障后的 `status` 只能看到停机后的 `actual=0/PWM=0`，不能代表触发瞬间的数据 |
| 下一步修改 | TIM2_CH1 PA15、TIM2_CH2 PB3 均配置为上拉输入；后续若线长或干扰仍明显，可外接 10k 上拉、优化走线，并记录稳定 PID 参数 |

### 第 5 次调试：限制正向目标下的负 PWM，消除反向刹车振动

| 项目 | 内容 |
|---|---|
| 日期 | 2026-06-11 |
| 测试目标 | 解决 `kp` 增大后电机一卡一卡、震动、超过目标后被反向拉住的问题 |
| 当前代码版本 | `Control_Update()` 中加入“正向目标时不允许 PWM 为负”的保护 |
| 硬件状态 | 编码器 TI12 解码、PA15/PB3 已配置上拉、双相接入正常 |
| 串口命令 | `set target uart`、`t=300`、`ki=0`、`kd=0`，旧讨论中围绕较大 `kp` 观察负 PWM 现象 |
| PID 参数 | 较大 `kp` 旧讨论数据，非最终稳定范围；最终以第 6 次真实测试为准 |
| 目标速度 | `target=300 RPM` |
| 实际现象 | 未限制负 PWM 时，`actual` 一旦超过 `target`，PID 会给负 PWM，电机表现为一卡一卡、震动，像被反向拉住；加入限制后，正向目标下的反向制动现象明显减少 |
| 串口输出摘录 | 旧现象示例：`target:300,actual:471,delta:11,PWM:-342,kp:2.00` |
| 问题判断 | 当前单向调速场景下，负 PWM 不是“轻轻减速”，而是反向制动。低分辨率测速下 `actual` 轻微超过目标就可能触发反向 PWM，导致正转被突然拉住 |
| 下一步修改 | 后续如需双向调速，再把“允许负 PWM”设计成按目标方向切换；当前单向场景先保持此保护 |

| 项目 | 内容 |
|---|---|
| 日期 | 2026-06-11 |
| 测试目标 | 用真实串口数据区分“程序异常”和“Kp 过大导致的启动冲击、卡顿、超速” |
| 当前代码版本 | 编码器 TI12 解码，PA15/PB3 已上拉，速度使用滑动窗口 |
| 硬件状态 | 电机、驱动、编码器反馈链路已能正常工作 |

| 目标速度 | ADC 给定约 `target=551~552 RPM` |
| 实际现象 | `kp=0.40` 时 PWM 约 220，电机不动或启动不足；`kp=0.50` 时出现一卡一卡，速度跳到目标以上后 PWM 变负；`kp=0.70` 时一启动就进入 `SPEED_OVER_LIMIT` |
| 串口输出摘录 | `kp=0.40`: `target:552,actual:0,delta:0,PWM:220,fault:NONE`；`kp=0.50`: `target:551,actual:685,delta:3,PWM:-67,fault:NONE`；`kp=0.70`: `enable:0,State:FAULT,target:551,actual:0,delta:0,PWM:0,fault:SPEED_OVER_LIMIT` |

| 下一步修改 | 调参时先固定目标源和目标值；保持 `ki=0,kd=0`，从不会卡顿的较小 Kp 开始；若要提高启动能力，优先考虑目标斜坡或死区补偿，而不是继续增大 Kp |

### 第 7 次调试：区分负 PWM 反拉和 PWM 太小顶不动

| 项目 | 内容 |
|---|---|
| 日期 | 2026-06-11 |
| 测试目标 | 判断 `kp=0.47` 附近的抖动是负 PWM 反拉，还是正向 PWM 太小导致电机顶不动 |
| 当前代码版本 | 正向目标下禁止负 PWM 的代码被注释，`PWM < 0` 会进入 `Motor_SetPWM()` |
| 硬件状态 | 编码器 TI12 解码，PA15/PB3 已上拉，速度使用滑动窗口 |
| 串口命令 | ADC 目标源下测试，`kp=0.47, ki=0, kd=0` |
| PID 参数 | `kp=0.47, ki=0.00, kd=0.00` |
| 目标速度 | ADC 给定约 `target=551~552 RPM` |
| 实际现象 | 一段时间内 PWM 一直为正但电机仍抖动，表现更像转不动或力不够；偶尔 `actual` 略高于 `target` 时出现很小的负 PWM，例如 `PWM=-2` |
| 串口输出摘录 | `target:552,actual:428,delta:0,PWM:58`；`target:551,actual:85,delta:1,PWM:219`；`target:552,actual:214,delta:-1,PWM:159`；`target:552,actual:557,delta:2,PWM:-2` |
| 问题判断 | 这不是自相矛盾，而是纯 P 控制在电机死区和低分辨率测速下形成的循环：`actual` 接近目标时误差小，P 输出变小，`PWM=58` 这类数值可能顶不动电机；速度掉下去后误差变大，PWM 又跳到 159 或 219；偶尔 `actual` 被滑动窗口估到目标以上，P 输出就会跨过 0 变成很小的负数 |
| 下一步修改 | 后续应把“正向目标禁止负 PWM”和“死区补偿/基础驱动力”分开处理。负 PWM 限制解决反向拉扯，死区补偿解决 PWM 太小顶不动；两者不是同一个问题 |

### 第 8 次调试：用启动助推解决静摩擦导致的随机启动失败

| 项目 | 内容 |
|---|---|
| 日期 | 2026-06-14 |
| 测试目标 | 区分“维持运行所需 PWM”和“从静止打破静摩擦所需 PWM”，并引入启动助推 |
| 当前代码版本 | 开始引入 `PWM_ff + PID` 前馈结构，并增加 `s_pwm_start_boost_active` / `s_pwm_start_boost_tick` 启动助推状态 |
| 硬件状态 | 编码器 TI12 解码，PA15/PB3 已上拉，电机和编码器反馈链路正常 |
| 串口命令 | 固定 PWM 开环测试，`kp=0, ki=0, kd=0`，多次从静止启动 |
| PID 参数 | `kp=0.000, ki=0.000, kd=0.000` |
| 目标速度 | ADC 目标约 `target=551 RPM`，开环测试时 target 只用于显示或计算前馈，不应参与 PID 输出 |
| 实际现象 | 使用开环点 `(actual=1328, PWM=220)` 和 `(actual=1242, PWM=210)` 估算得到 `kS≈66, kV≈0.12`，则 `target=551` 时 `PWM_ff≈132`。但 `PWM=132/150/170` 从静止时可能无法启动；当手动给电机一个外力后，同样 PWM 可以维持转动 |
| 串口输出摘录 | 示例：`PWM:170, actual:0, delta:0, kp:0.000, ki:0.000, kd:0.000`。开环拟合点：`PWM=210 -> actual≈1242`，`PWM=220 -> actual≈1328` |
| 问题判断 | `kS + kV * target` 得到的是“已经转起来后的维持 PWM”，不能保证从静止打破静摩擦。启动失败不是 PID 计算错误，而是静摩擦、齿轮啮合位置、转子位置等导致的启动滞回 |
| 下一步修改 | 将控制策略拆成三部分：启动助推负责打破静摩擦；`PWM_ff = kS + kV * target` 负责维持大概速度；PID 只负责误差修正。`run=1` 时打开启动助推，若 `encoder_delta != 0` 或超时则退出助推，并切回正常 `PWM_ff + PID` |

## 记录原则

- 不要把示例数据当成真实结果。
- 每次只验证一个目标，例如“串口命令是否生效”或“Kp 增大后响应是否变快”。
- 每次记录至少保留一段串口输出摘录。
- 如果现象不确定，就写“不确定”，不要补成看起来很完整的结论。
