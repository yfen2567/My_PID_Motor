# PID 调参记录

PID 调参记录用于说明每一次参数修改对应的实际现象。以后复盘或面试时，要能说清楚：为什么 Kp 这样选，为什么 Ki 后加，哪些参数会导致波动。

## 1. 调参原则

推荐顺序：

```text
Ki = 0
Kd = 0
先只调 Kp
确认方向和响应没问题后，再加少量 Ki
Kd 暂时可以不急着用
```

不要一开始同时调 Kp、Ki、Kd，否则很难判断现象是谁造成的。

## 2. 调参前必须先确认

| 检查项 | 为什么重要 | 状态 |
|---|---|---|
| `run=0` 能让电机停止 | 避免调参时失控 | 已确认 |
| `target` 能被串口命令修改 | 否则调参没有目标依据 | 已确认，调参时先 `set target uart` |
| `actual` 会随电机转动变化 | 否则 PID 没有有效反馈 | 已确认，编码器接线和 PA15/PB3 上拉后正常 |
| `pwm` 会随误差变化 | 判断 PID 是否在工作 | 已确认 |
| 编码器方向正确 | 方向反了会导致越调越错 | 已确认，A/B 相反接时会出现正向 PWM 下负速度 |

## 5. 本次实测结论

- 调参前必须先固定目标来源。串口调参时先发 `set target uart`，否则 ADC 会周期刷新 `target`。
- `ki`、`kd` 初期先保持 0。尤其 `kd=0.10` 在当前离散速度反馈下很容易把 PWM 打满。
- 当前编码器低速分辨率有限。10ms 下 `delta=1` 对应约 `214RPM`，50ms 下 `delta=1` 对应约 `42RPM`，所以低速调参要结合 `delta` 看。
- 只看故障后的 `status` 不够。进入故障后 `actual/PWM` 可能已经被清零，真正触发瞬间的数据会丢失。
- PA15/PB3 上拉前的数据不能作为 PID 依据；上拉后再重新记录稳定参数。
- 当前项目以正向目标为主时，负 PWM 往往表现为反向刹车，不是平滑减速。`actual` 略超目标就给负 PWM，会把电机拉成震动；限制正向目标下的负 PWM 可以减少反向制动，但不能把过大的 Kp 变成稳定参数。
- ADC 目标约 `551 RPM` 的真实测试中，`kp=0.40` 启动不足，`kp=0.50` 开始卡顿，`kp=0.70` 触发 `SPEED_OVER_LIMIT`。这组数据应作为后续调参的主要依据。
- `kp=0.47` 的数据说明，抖动不一定都来自负 PWM 反拉。即使 PWM 多数时间为正，只要接近目标后 P 输出降到电机死区附近，也会出现“推一下、松一下”的抖动。
- `PWM_ff = kS + kV * target` 是稳定运行前馈，不负责保证从静止启动。若为了打破静摩擦而把 Kp 调大，稳定运行时容易重新出现过冲和抖动；启动应交给启动助推，PID 只做误差修正。

## 6. 观察重点

每次调参后重点观察：

- `actual` 是否接近 `target`。
- `actual` 是否来回振荡。
- `pwm` 是否长期接近最大值。
- `pwm` 很大但 `actual` 不变时，要先怀疑硬件或编码器链路，不要急着调 PID。
- 加 Ki 后误差是否变小。
- 加 Ki 后是否出现积分过大导致的恢复慢或波动。
- Kd 初期可以先保持 0，等 P/PI 能跑通后再考虑。

## 7. 建议调参流程

```text
1. run=0，确认电机能停止。
2. 发送 `set target uart`，设置 target 为较小值，例如 `t=300`。
3. 设置 ki=0，kd=0。
4. 从较小 Kp 开始，观察 actual 是否能跟随 target。
5. Kp 增大到响应较快但不明显振荡。
6. 少量增加 Ki，让长期误差变小。
7. 每改一次参数，就在“我的正式调参记录”里新增一行。
```

真实调参时，把串口输出中的 `source,target,actual,delta,pwm,kp,ki,kd,fault` 连同现象一起记录。若出现故障，先确认是否为编码器接线、上拉、目标源或方向问题，再判断 PID 参数。
