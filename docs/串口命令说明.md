# 串口命令说明

## 1. 命令格式

当前项目使用简单文本命令，建议每条命令以换行结束。串口助手发送时可以选择追加 `\r\n` 或 `\n`。

示例：

```text
run=1
t=500
kp=0.5
```

当前接收逻辑是一字节一字节进入 UART 接收完成回调，普通字符会进入接收缓存，遇到 `\r` 或 `\n` 后认为一条命令结束，再交给 `Uart_Task()` 解析。

## 2. 支持的命令

| 命令 | 参数类型 | 作用 | 示例 | 备注 |
|---|---|---|---|---|
| `run=1` | 开关量 | 开启闭环 | `run=1` | 开启后 PID 输出控制电机 |
| `run=0` | 开关量 | 停止电机 | `run=0` | 停止时应让 PWM 指令为 0，并复位 PID 历史状态 |
| `t=<speed>` | 整数 | 设置目标速度 | `t=500` | 单位按项目内部定义，当前可理解为编码器计数变化率相关量 |
| `kp=<value>` | 浮点数 | 设置 Kp | `kp=0.5` | 初期建议先只调 Kp |
| `ki=<value>` | 浮点数 | 设置 Ki | `ki=0.1` | Kp 可用后再加少量 Ki |
| `kd=<value>` | 浮点数 | 设置 Kd | `kd=0` | 初期可先不用 Kd |
| `rst` | 无 | 复位 PID 状态 | `rst` | 用于重新开始调试 |

## 4. 调试建议

每次输入命令后，观察串口输出中的这些字段：

```text
ms,run,target,actual,pwm,kp,ki,kd
```

重点看：

- `target` 是否变成设定值。
- `actual` 是否跟随目标变化。
- `pwm` 是否长期接近最大输出。
- `run` 是否和命令一致。
- `kp / ki / kd` 是否和刚发送的调参命令一致。
