## 3. 负载扰动汇总结论

测试条件为当前硬件、当前固件、UART 目标源、`kp=0.05, ki=0, kd=0` 和人工轻载扰动。

| 测试 | target | 单项结论 | 主要趋势 |
| --- | ---: | --- | --- |
| LD-001 | 500 | PASS（初步通过） | actual 均值下降 29.8，PWM 均值上升 1.4；释放确认后回到 baseline 附近 |
| LD-002 | 600 | PASS（轻载趋势初步通过，扰动幅度有限） | actual 均值下降 16.8，PWM 均值上升 0.9；释放确认后回到 baseline 附近 |
| LD-003 | 700 | PASS（轻载趋势初步通过，扰动幅度有限） | actual 均值下降 19.5，PWM 均值上升 0.9；释放确认后回到 baseline 附近 |

三组正式日志均记录到 `fault:NONE`；stop 后均为 `enable=0、State=IDLE、PWM=0`；`get fault` 均返回 `FAULT_SNAPSHOT_NONE`。按当前负载扰动计划，LD-001/002/003 负载扰动证据链已完成，可作为 v2.1 候选证据的一部分，不追加 LD-004。

该结论只表示 **人工轻载条件下观察到基本补偿趋势**，不是精确恢复时间、控制带宽、频响、转矩或工业级稳定性验证。

## 4. 已有证据支持的内容

- LD-001/002/003 均有正式原始日志、完整关键 MARK、释放确认段和单项报告；
- 三档施加载荷后 actual 均值下降，PWM 均值上升，变化方向符合软件补偿预期；
- 释放确认后 actual/PWM 回到各自 baseline 附近并继续运行；
- 三组状态输出均未记录 fault，stop 后软件状态与 PWM 命令归零；
- 三组无快照查询均返回可解释的 `FAULT_SNAPSHOT_NONE`；
- README、验证计划、日志字段字典、负载扰动计划、汇总报告和 checklist 均已存在。

## 6. 已知异常与保留记录

- **LD-001 第一次释放时序问题**：首次尝试的人工释放时机与 MARK 边界不可靠，已保存原始日志和归档报告，不纳入正式统计；
- **LD-001 stop 后单点 `actual=-85`**：PWM 已为 0，后续 actual 回到 0；异常点保留，现有日志不足以确定根因；
- **LD-003 启动瞬态 `actual=1200`**：出现在 `run=1` 后、baseline 前，后续回到正常范围；保留但不纳入 baseline、load 或 release 均值；
- 个别异步 status 在文件顺序上可能交织；正式统计依照 MARK 和固件 `ms` 分段，没有据此推断 MCU 复位。

这些异常没有被删除或简单归因为代码问题。

## 7. 不能外推的边界

本轮结果不能证明：

- 精确恢复时间、控制带宽、频响或严格动态性能；
- 长期可靠性、工业级稳定性或全部异常场景覆盖；
- 其他电机、驱动板、供电、负载或 PID 参数具有相同结果；
- 三次人工负载的大小和持续时间完全一致；
- 物理 PWM 波形、电流和真实转矩符合软件字段；
- 真实 fault snapshot 有快照分支可用。
