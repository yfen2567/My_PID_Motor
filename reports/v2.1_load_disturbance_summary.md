# My_PID_Motor v2.1 负载扰动测试汇总报告

## 1. 汇总结论

LD-001、LD-002、LD-003 三档负载扰动测试均已完成正式重测或正式测试，并分别形成原始日志和单项报告：

- LD-001（target=500）：PASS（初步通过）；
- LD-002（target=600）：PASS（轻载趋势初步通过，扰动幅度有限）；
- LD-003（target=700）：PASS（轻载趋势初步通过，扰动幅度有限）。

三组正式日志均记录到：施加人工轻载后 actual 均值下降、PWM 均值上升；`RELEASE_LOAD_CONFIRMED` 后 actual/PWM 回到各自 baseline 附近；全程 `fault:NONE`；stop 后 `enable=0、State=IDLE、PWM=0`；`get fault` 返回 `FAULT_SNAPSHOT_NONE`。

按当前 v2.1 负载扰动测试计划，500/600/700 三档已经具备相互独立的正式证据，可以作为本阶段收口依据。该结论是**人工轻载趋势验证**，不是精确动态性能验证，也不代表长期可靠性或工业级稳定性已经确认。

## 4. 趋势对比

三组正式结果表现出相同方向的定性趋势：

1. 施加人工轻载后，actual 均值均下降：500/600/700 档分别下降 29.8、16.8、19.5；
2. 同期 PWM 均值均上升：分别提高 1.4、0.9、0.9，说明软件输出命令在尝试补偿速度下降；
3. 释放确认后，三组 actual 和 PWM 均回到各自 baseline 统计范围附近；
4. 三组运行与恢复阶段均保持 `enable=1、State:RUN、source:Uart`，没有出现持续 `actual=0、delta=0`；
5. 三组全部 status 均记录为 `fault:NONE`，stop 后均为 `enable=0、State=IDLE、PWM=0`；
6. 三组 `get fault` 均返回 `FAULT_SNAPSHOT_NONE`。

LD-002 和 LD-003 的均值变化较小，且小于一个约 42.9 RPM 的测速量化单位；它们的结论必须保留“扰动幅度有限”边界。由于人工负载大小和操作力度没有量化，不能用三组下降幅度对不同 target 的控制性能进行排名。

## 5. 异常与特殊说明

### 5.1 LD-001 第一次时序问题归档

第一次 LD-001 尝试保存在：

- `logs/v2.1_LD-001_attempt1_timing_issue_raw.txt`
- `reports/v2.1_LD-001_attempt1_timing_issue_report.md`

该次脚本 MARK 与人工释放动作的时序不能作为可靠恢复边界，日志还记录了释放标记后 `actual=0、delta=0、PWM=140` 持续到 stop 的现象。该异常没有删除或改写，但由于人工时机问题，不纳入本汇总的正式数值。LD-001 正式统计使用带 `RELEASE_LOAD_CONFIRMED` 的重测日志。

### 5.2 LD-001 stop 后单点 actual=-85

LD-001 正式日志在 stop 已执行后记录一条：

```text
ms:29697,enable:0,State:IDLE,target:500,actual:-85,delta:0,PWM:0,fault:NONE
```

后续 status 均为 actual=0，PWM 始终为 0。该单点已保留，不影响 stop 后软件输出归零结论；现有日志不足以把它简单归因为代码、机械回弹或测速窗口残留。

### 5.3 LD-003 启动瞬态 actual=1200

LD-003 在 `run=1` 后、`BASELINE_START` 前记录：

```text
ms:12468,enable:1,State:RUN,target:700,actual:1200,delta:5,PWM:115,fault:NONE
```

进入 baseline 后 actual 回到 642～728，未持续出现 1200。该启动瞬态已保留，但依据阶段规则没有纳入 baseline、load 或 release 均值，也没有用它直接判定负载扰动失败。现有串口日志不能确认其物理转速真实性或根因。

### 5.4 日志顺序与采样边界

三组日志初始化阶段均存在周期 status 与主动查询交织，个别固件 `ms` 在文件顺序上轻微倒置；正式阶段没有 MCU 复位证据。统计同时包含周期 status 和主动查询结果，样本不是严格等间隔独立采样，因此均值仅用于趋势描述，不能用于计算精确恢复时间。

## 6. 阶段结论边界

### 本阶段能证明

- 在当前硬件、当前固件、`kp=0.05、ki=0、kd=0` 和当前人工轻载条件下，target=500/600/700 三档均观察到基本轻载扰动响应趋势；
- 施加轻载后，actual 均值下降且 PWM 均值上升，变化方向符合补偿预期；
- `RELEASE_LOAD_CONFIRMED` 后，系统能够恢复到各自 baseline 附近并继续运行；
- 三组均未记录 fault 或持续停速；
- 三组 stop 后的软件状态和 PWM 命令均归零；
- 三组无历史快照查询均得到可解释的 `FAULT_SNAPSHOT_NONE`。

### 本阶段不能证明

- 精确恢复时间；
- 控制带宽或频响；
- 长期可靠性；
- 工业级稳定性或功能安全；
- 其他电机、驱动板、供电、负载或 PID 参数下具有相同结果；
- 三次人工负载大小、持续时间和动作完全一致；
- PA8 物理 PWM 波形、方向脚、驱动电流、供电压降和真实转矩特性；
- 实际 fault snapshot 有快照分支已经验证；
- 启动瞬态和 stop 后单点速度异常的根因已经确定。

三份原始日志均未记录具体 LLCOM 脚本文件名、固件 commit 和构建产物 hash。这不影响本轮串口趋势记录本身，但仍是 v2.1 release 证据追溯需要补齐的内容。

## 7. 是否需要继续负载扰动测试

按当前 v2.1 负载扰动测试计划，LD-001/002/003 可以收口，不建议继续追加 LD-004。

三组均有正式原始日志、完整关键 MARK、释放确认段、单项报告和可解释的 stop/get fault 结果；没有一组需要因证据不足而复测。若未来需要精确恢复时间、定量负载或控制带宽，应重新定义测试装置和判据，而不是继续追加同类人工轻载编号。
