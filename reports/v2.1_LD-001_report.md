## 10. 异常与边界说明

### 本次测试能证明

- 在本次硬件与人工轻载条件下，`target=500、Kp=0.05、Ki=0、Kd=0` 能保持 RUN；
- 施加轻载后 actual 有下降，PWM 出现同方向补偿；
- 释放确认后 actual 和 PWM 回到扰动前统计范围附近；
- 本次过程中没有记录到 fault、持续停速、串口中断或 MCU 复位；
- stop 后软件状态和 PWM 命令安全归零；
- 无历史 fault snapshot 时查询结果为 `FAULT_SNAPSHOT_NONE`。

### 本次测试不能证明

- 人工负载的精确大小、转矩或动作重复性；
- 精确恢复时间、控制带宽、频响或严格动态性能；
- PA8 实际 PWM 波形、方向脚、电机电流、供电压降或编码器 A/B 相电气质量；
- 其他电机、驱动板、供电、负载和 PID 参数下具有相同结果；
- target=600/800 的 LD-002、LD-003 已经通过；
- 长期可靠性或全部异常场景已经覆盖。

原始日志未记录具体脚本文件名、固件 commit 和构建产物 hash；这些信息在 v2.1 release 证据链中仍需补齐。日志开头两条周期 status 的固件 `ms` 在文件顺序上出现 `13740` 后接 `13528`，位于正式基线之前，可能与异步接收合并顺序有关；本报告没有用这两条数据计算负载阶段结果，也不能仅凭此判断通信或 MCU 异常。

LD-001 只能覆盖低目标速度。若要完成既定的三档负载验证，仍需分别执行 LD-002 和 LD-003，不能由本次结果外推。
